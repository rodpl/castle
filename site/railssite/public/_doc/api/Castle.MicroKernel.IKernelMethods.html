The methods of the IKernel interface are listed below. For a complete list of IKernel interface members, see the IKernel Members topic.

Public Instance Methods

AddChildKernel Support for kernel hierarchy
AddComponentOverloaded. Adds a concrete class and an interface as a component
AddComponentWithPropertiesOverloaded. Adds a concrete class and an interface as a component and specify the extended properties. Used by facilities, mostly.
AddCustomComponent Adds a custom made ComponentModel. Used by facilities.
AddFacility Adds a IFacility to the kernel.
AddSubSystem Adds (or replaces) an ISubSystem
CreateComponentActivator Constructs an implementation of IComponentActivator for the given ComponentModel
GetAssignableHandlers Return handlers for components that implements the specified service. The check is made using IsAssignableFrom
GetHandlerOverloaded. Returns the IHandler for the specified service.
GetHandlers Return handlers for components that implements the specified service.
GetSubSystem Returns an implementation of ISubSystem for the specified key.
HasComponentOverloaded. Returns true if the specified service was registered
ReleaseComponent Releases a component instance. This allows the kernel to execute the proper decomission lifecycles on the component instance.
RemoveComponent Returns true if the specified component was found and could be removed (i.e. no other component depends on it)
