AppDomainConfigurationStore Looks for a section 'castle' in the standard Configuration scheme. It then deserialize the nodes to IConfiguration implementation. 
<configuration>

  <configSections>
    <section name="castle"
      type="Castle.Windsor.Configuration.AppDomain.CastleSectionHandler, Castle.Windsor" />
  </configSections>
  
  <castle>
    <facilities>
      <facility id="myfacility">
    
      </facility>
    </facilities>
  
    <components>
      <component id="component1">
    
      </component>
    </components>
  
  </castle>
  
</configuration>
CastleSectionHandler Implementation of IConfigurationSectionHandler. Do not support configuration inheritance.
